# Семинар №4 — Создание классов и коллекций

## Описание

Проект выполнен по дисциплине **«Архитектура автоматизированных систем обработки информации и управления»**.

Тема семинара: **создание классов и коллекций на C#**.

В рамках работы реализовано консольное приложение, которое демонстрирует:

- создание системы классов «Геометрические фигуры»;
- использование наследования и абстрактных классов;
- реализацию разреженной матрицы на основе стандартной коллекции `Dictionary`;
- создание собственного однонаправленного списка;
- реализацию стека на основе собственного списка;
- сортировку элементов списка по площади фигуры.

## Реализованные классы

### Геометрические фигуры

В проекте реализована система классов:

- `Figure` — абстрактный базовый класс фигуры;
- `Rectangle` — прямоугольник;
- `Square` — квадрат;
- `Circle` — круг.

Каждая фигура имеет свойство `Area`, которое возвращает площадь фигуры.

Также класс `Figure` реализует интерфейс `IComparable`, что позволяет сравнивать фигуры по площади.

### Разреженная матрица

Класс `Matrix<T>` реализует разреженную матрицу.

Для хранения элементов используется словарь:

```csharp
Dictionary<(int x, int y), T>
```

Ключом является кортеж координат ячейки матрицы, а значением — объект, который хранится в этой ячейке.

Для работы с пустыми элементами используется интерфейс:

```csharp
IMatrixCheckEmpty<T>
```

Для геометрических фигур реализован класс `FigureMatrixCheckEmpty`, который считает пустым элементом значение `null`.

### Собственный список

Класс `SimpleList<T>` реализует простой однонаправленный список без использования стандартных коллекций.

Возможности списка:

- добавление элемента в конец списка;
- получение элемента по индексу;
- перебор элементов через `foreach`;
- сортировка элементов методом быстрой сортировки.

Для хранения элементов списка используется класс `SimpleListItem<T>`, который содержит данные и ссылку на следующий элемент.

### Стек

Класс `SimpleStack<T>` реализует стек на основе собственного списка.

Стек работает по принципу:

```text
LIFO — Last In, First Out
```

То есть последний добавленный элемент извлекается первым.

## Основные идеи работы

В проекте используется наследование: классы `Circle`, `Rectangle` и `Square` наследуются от абстрактного класса `Figure`.

Разреженная матрица реализована поверх стандартной коллекции `Dictionary`. Такой подход позволяет хранить только заполненные элементы матрицы и не выделять память под пустые ячейки.

Собственный список и стек реализованы без использования стандартных коллекций. Это показывает принцип работы пользовательских структур данных.

Сортировка списка выполняется по площади фигур. Для этого класс `Figure` реализует интерфейс `IComparable`.
